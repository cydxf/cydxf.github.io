To think like a computer

发表于 2024-03-28 更新于 2024-03-30 阅读次数： 860 1 分钟

有本书叫《How to think like a computer scientist》，我看完了C version之后就好奇去网上查了一下它的书评，结果却意外看到了一句令我印象至深的话

To think like a computer

那怎么像计算机一样思考呢
首先，计算机会思考吗
你不过是台机器，满脑子都是如何解决人类世界的各种问题

To talk with a computer

计算机，你会什么

我会运算，我懂控制，我能存储，我欣然接受输入，也积极输出
你发现不了的问题，我却能立马指出

对欸，你是怎么发现我的问题的，你是怎么解答我的疑惑的，我天马行空的想象你都能做出回应。你仅靠运算、控制、存储、输入、输出就能实现如此多的功能，那岂不是在说，{运算、控制、存储、输入、输出}就是世界上一切算法的完备集

不错，所以你愿意聆听我的想法吗

洗耳恭听

首先我想问你，你是怎么看懂我们的程序的

首先，要明确一点，我是看不懂你们的语言的。我能看懂的只有二进制代码，也就是你们俗称的0和1，当然，这些0和1往往对应着实体，比如高低电平、磁极的正负、光盘的凹凸。你们所写的C++等语言，先经过解释（一行一行读）或编译（一次性读完）转化为低级机器代码汇编代码，汇编代码再链接得到可进行的二进制文件

其实最令我惊奇的，不是你能将我们的语言转化为二进制语言，而是你能读懂二进制语言

布莱叶文字不是一样的原理么，盲人不也是通过纸张的凹凸解读这个世界

盲文相当于是一个二维二进制文字，那你的二进制读取的时候是一维度读取的啊，你是怎么短句的，哦，如果是C++读到；就断句。那还是不对啊，就算你读懂了我写的是hello world,那你是怎么输出到屏幕上的呢

你写的并不是hello world,而是cout<<”hello world”;这个cout<<我也读不懂的，我是怎么区分cout这个指令与hello world这个元语言的，因为cout不是cout，而是std::cout

我还想问，为什么我在5G的时代，体验着4G的冲浪速度，3G的加载速度，2G的下载速度，这是你的多态性吗
