Visual Studio vs VScode

发表于 2023-08-31 阅读次数： 823 1 分钟

今天在xhs看到有人讨论Visual Studio和VScode哪个好的问题，这个我深有体会，毕竟我两个都体验过。

1.Visual Studio

作为一个Freshman，我刚开始学编程的时候急于找到一个可用的编译器，无意间了解到了Visual Studio。当时我尝试了Linux的gcc和vs，一时觉得vs实在太好用了。gcc要cd ls ./，而且终端和编辑器还是独立的，而vs开箱即用，点击即可呈现结果，操作简单易懂，还能实时显示错误。

2.VScode

后来我了解到VScode，但听说要配置环境便被劝退了，还在留恋VS开箱即用的好。但后来在学长的推荐下开始使用VScode，才发现配置环境其实没有这么难，而且也是必经之路。掌握了VScode之后，我才发现VScode真香，我之前一直想把自己的代码记录形成系统，但VS的Project会自动建立，对于连文件路径都还不太了解的小白来说，找到VS的Project并进行修改其实有空气墙，而选中文件后用VS打开更一无所知。而VScode则不同，它初始化就是要人手动建立文件夹和文件，不知不觉中便形成了系统。而且它的轻便让动辄好几个G的VS甘拜下风。VScode还有良好的插件生态，可以拥有属于自己的个性化的编辑器。于我而言，由VS转到VScode也开启了新世界的大门。而VScode的兼容性让我叹为观止，C、Python、yml、html、pdf、word、txt、log、markdown、dll、…你能想到的所有文件几乎都可以用VScode打开，例如我这篇博文就是在VScode中写的。而且VScode还受到cs61a和cs50的官方推荐。因为VScode，我学会了环境配置。通过VScode又了解到Windows Powershell，开始知道cd mv ls ./ rm等命令提示符，开始使用cmd以及Git，而现在，我又准备开始入手Vim。

VScode真香，跪拜，是我唯一能为你做的事了。
